Gaps & suggested screens

Since the June 29 audit, 16 new screens were added: Conversations and MLS search on both surfaces, Documents stub, Contact Detail, and 5 auth screens on each platform. The 7 P3 brief items remain open. A second block below lists 8 suggested screens not currently in the brief that would round out the product.

P3 — Design-brief features, not yet started

  • 01
    Dark mode
    design.md §2 states: "Dark mode is first-class. Agents use the app in the car at night." Neither surface has a dark theme. The Desktop Settings screen has a Theme toggle (Light/System/Dark) that does nothing.
    Both
  • 02
    Desktop — Command Palette (⌘K)
    design.md §9.11 specifies a ⌘K overlay with two groups: Actions (New Lead, New Contact, etc.) and grouped search results (Contacts / Properties / Offers / Notes). The topbar search field exists but there is no keyboard shortcut handler.
    Desktop
  • 03
    Desktop — Global Search results overlay
    The topbar search field in desktop/shell.jsx is decorative only. Typing does nothing. Needs a results dropdown/overlay grouped by type (Lead, Client, Property, Offer, Note) — similar to mobile's SearchScreen.
    Desktop
  • 04
    Desktop — Offers: Side-by-side comparison view
    design.md §11.8: select 2+ offers on the same property → vertical column comparison layout showing Price / Deposit / Conditions / Closing / Deadline / Terms side by side. The Offers table has no multi-select and no comparison view.
    Desktop
  • 05
    Desktop — Offer Detail: Negotiation timeline sparkline
    design.md §11.9: a vertical history of counters, revisions, and acceptances with the offer amount charted over time as a small sparkline. The Timeline tab currently shows a plain activity feed with no price chart.
    Desktop
  • 06
    Desktop — Calendar: Conflict detection banner
    design.md §11.10: when two appointments overlap, a banner appears at the top of the week view with a one-click Reschedule action. Thursday Apr 17 has overlapping events that could demonstrate this.
    Desktop
  • 07
    Desktop — Tasks: Bulk-edit table view + mini-calendar scheduler
    design.md §11.11: (1) a bulk-editable table view toggle and (2) a today's task scheduler mini-calendar above the list for dragging unscheduled tasks onto time slots. Neither exists.
    Desktop

Suggested — Not in brief; recommended additions

  • S1
    Documents vault — full build
    The mobile stub shows the intent: a file browser linked to clients and listings, with category folders (Listing Agreements / Offer Docs / Inspection Reports / Client Files / MLS Sheets), upload action, and document preview. The Desktop currently surfaces Documents only inside record detail tabs; a top-level Documents screen is missing on both surfaces.
    Both
  • S2
    Analytics & reporting dashboard
    A dedicated reporting screen: GCI to date vs. annual goal, deal volume by month (sparkline or bar), pipeline conversion funnel (Lead → Client → UC → Closed), average DOM, and source attribution breakdown. The dashboard stat chips hint at this data but there is no drill-down report view.
    Both
  • S3
    Commission calculator / deal P&L
    Per-deal screen: sale price, commission rate, splits, referral fee, brokerage desk fee → net take-home. Useful for both previewing an offer and reconciling a closed deal. Could live as a tab inside Offer Detail or as a standalone calculator modal.
    Both
  • S4
    Showing feedback collection
    After a showing event, a sheet or screen to log buyer agent feedback: interest level, price reaction, specific objections, likelihood to offer. Currently feedback is free-form in Notes with no structure. Could trigger automatically from a completed showing event.
    Both
  • S5
    Open house check-in sheet
    A dedicated open house mode: fullscreen sign-in form (name, phone, email, "working with an agent?"), live visitor count, auto-create Lead records for each sign-in. Mobile is the natural surface — the agent hands their phone to visitors.
    Mobile
  • S6
    CMA / comparative market analysis viewer
    A read-only CMA view attached to a Seller client or Listing: comparable sales table (address, sale price, sqft, $/sqft, DOM, close date), price range summary, and suggested list price range. Feeds the listing presentation. Desktop-primary but useful as a reference on mobile during listing consults.
    Both
  • S7
    Contact import (CSV upload)
    Onboarding and ongoing import flow: drag-and-drop CSV, column mapping step (Name / Phone / Email / Role / Source), duplicate detection, import summary. Desktop modal. An empty Contacts or Leads list with a prominent "Import contacts" CTA would surface this naturally.
    Desktop
  • S8
    Annual goal tracker
    Set and track GCI goal, deal count target, and listing target for the year. Shows progress rings or step indicators alongside pipeline stats. Could live inside Settings or as a pinnable widget on the dashboard. Gives the agent a north star number visible every day.
    Both
